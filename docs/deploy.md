# Deployment Walkthrough

This is the bootstrap path for issue #9. A fuller deployment guide lands in issue #17.

## Prerequisites

- Terraform has provisioned the EKS cluster from `terraform/`.
- Your kubeconfig points at the private EKS API endpoint from an allowed network path.
- `kubectl` is installed and configured for the target cluster.

## Bootstrap Argo CD

Argo CD is installed from the pinned upstream manifest at `argoproj/argo-cd` tag `v2.13.3` and hardened with local overlays in `gitops/bootstrap/`.

```bash
kubectl apply -f gitops/bootstrap/argocd-namespace.yaml
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/v2.13.3/manifests/install.yaml
kubectl apply -f gitops/bootstrap/argocd-install.yaml
kubectl -n argocd rollout status deploy/argocd-server
kubectl -n argocd get pods
```

The bootstrap keeps `argocd-server` as `ClusterIP`, so the UI is not exposed through a public LoadBalancer. Reach it through a local port-forward:

```bash
kubectl -n argocd port-forward svc/argocd-server 8080:443
```

Open `https://localhost:8080`.

## Initial admin access

The initial admin secret is generated by Argo CD during install. Retrieve it once, rotate it immediately, and then disable or replace local admin access with SSO in a follow-up ticket.

```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
```

The local RBAC overlay sets `policy.default: role:readonly`, disables anonymous access, and denies remote exec through Argo CD. Platform administrators should be mapped through SSO groups before production use.
